# 错题伙伴（暂定名）微信小程序 · 产品规格文档（Spec v0.2）

> 状态：草稿 / 待评审
> 范围：本文档只描述「做什么」和「为什么」，不含实现代码。等「请开发」指令后再进入开发。
> 适用平台：微信小程序（孩子端为主，家长端轻量）。

---

## 1. 背景与问题

家长面对孩子的校内错题，目前只能「直接讲」或「现场指导」。结果是：

- 讲完当下会了，但**没有内化**。考试或几天后遇到**同一题 / 同一类题又错**。
- 家长焦虑：不知道孩子到底有没有掌握这个知识点。
- 孩子**不主动复盘错题**，错题本沦为摆设；复习全靠家长催。

核心矛盾不是「不会讲」，而是：

1. **遗忘**——讲过的没有在正确的时间点重复，知识点没沉淀。
2. **缺乏理解检验**——"听懂了"不等于"会了"，没有客观验证。
3. **驱动力错位**——上传和复习的动力来自家长，不是孩子自己。

## 2. 产品定位与一句话价值

> 一个让孩子**愿意自己动手**整理错题、并用**费曼讲解 + 间隔复习**真正掌握背后知识点的微信小程序。

产品足够「轻」：核心闭环就是「拍错题 → 讲一遍 → 被测一道类似题 → 到点再来复习」。

## 3. 目标与非目标

### 3.1 目标（v1）
- 让孩子能用**最低操作成本**（拍照 + 语音）把一道错题录入。
- 用**费曼学习法**让孩子「讲出来」，暴露真实理解程度。
- 用**艾宾浩斯遗忘曲线**安排复习节奏，到点提醒孩子回来。
- 用**联网搜索 + 题库**生成同类题，检验是否真正掌握知识点。
- 让孩子**主动**使用，而不是家长代劳（见第 6 章，重点）。

### 3.2 非目标（v1 明确不做）
- 不做完整的「网课 / 系统授课」。我们只围绕**孩子自己的错题**。
- 不做大而全的全学段题库。**v1 聚焦上海小学**（数学优先，详见第 9 章）。
- 不做家长「布置作业 / 监控孩子」的强管控工具——那会破坏孩子主动性。
- v1 不做社交/排行榜的真人 PK（避免攀比与隐私风险），仅做个人成长激励。

## 4. 目标用户与角色

| 角色 | 描述 | 在产品中的位置 |
|------|------|----------------|
| 孩子（主角）| 上海小学生，约 7–12 岁，可能不会熟练打字 | 主要使用者，主操作端 |
| 家长（辅助）| 焦虑、希望孩子掌握知识点，愿意花钱 | 付费方 + 轻度查看，**不代劳录入** |
| 老师（未来）| 班级层面 | v1 不涉及，预留 |

设计原则：**孩子是第一用户，家长是第二用户。** 任何让家长更省心、却削弱孩子主动性的功能，都要谨慎。

## 5. 学习方法论（产品的理论内核）

### 5.1 费曼学习法（理解检验）
费曼的精髓是「能用简单的话讲给别人听 = 真懂」。在产品中落地为：

1. 录入错题后，让孩子**语音讲解**："这道题你会怎么做？讲给我听。"
2. 系统（AI）扮演「不太懂的同学/小伙伴」，根据孩子的讲解：
   - 找出讲解里**跳步、说不清、概念错误**的地方；
   - 用追问的方式让孩子补全（"你说要先通分，为什么要通分呀？"）。
3. 形成一个**理解度评分**和「卡点」清单，作为后续复习与测试的依据。

> 关键体验：AI 是「比孩子还笨、需要被教会的小伙伴」，而不是「考孩子的老师」。降低压迫感，激发"我来教你"的表达欲。

### 5.2 艾宾浩斯遗忘曲线（复习节奏）
每道掌握中的错题进入**间隔复习队列**，默认节点（可调）：

```
第 1 天（录入当天）→ 第 2 天 → 第 4 天 → 第 7 天 → 第 15 天 → 第 30 天
```

- 每个复习节点：出一道**同类题**让孩子做 + 简单复述知识点。
- **答对 + 讲清**：进入下一个更长的间隔。
- **答错 / 讲不清**：间隔**回退**（重置到较早节点），并标记知识点为「未掌握」。
- 到点用**温和的提醒**（见 6.4）召回孩子，而不是家长催。

### 5.3 知识点为中心
错题不是孤立的，每道题打上**知识点标签**（如"分数加减·异分母通分"）。同一知识点的多道错题聚合，帮助识别"反复错的薄弱点"。

## 6. ⭐ 核心设计：如何让孩子「自己愿意」用？

这是本产品成败的关键，单列一章。把"孩子主动"拆成三个心理阶段来设计。

### 6.1 录入要"零摩擦"（让上传比不上传还轻松）
- **拍照即录入**：打开就是相机，拍下错题，框选自动锁定题目区域（见 7.1），不用打字。
- **语音代替打字**：所有需要文字输入的地方都支持语音（见 7.2）。
- **3 步内完成**：拍 → 框 → 确认。不强制立刻讲解，可"先存起来晚点搞"。
- **降低心理负担**：不叫「错题」（带责备感），用中性/正向词，如「我的题库」「待解锁的题」。

### 6.2 把"复习"变成"游戏"（即时反馈 + 收集欲）
- **错题=可收集的卡片/怪物**：每道录入的题是一只"卡住知识点的小怪兽"，讲清楚 + 测试通过才能"驯服/点亮"它。利用收集癖。
- **理解能量条**：每只怪兽有 0→100 的能量条，随着一次次复习通过而填满；填满 = 这个知识点真正掌握，获得徽章。
- **即时正反馈**：讲解后立刻给鼓励性的、具体的反馈（"你把'为什么通分'讲清楚了，比上次进步！"），而不是冷冰冰的对错。
- **连续打卡（streak）**：连续天数的轻量激励，但**断签不惩罚、不羞辱**（避免压力反噬，断了温柔重启）。

### 6.3 给孩子"掌控感"与"主角感"
- **AI 是孩子要"教会"的小伙伴**：孩子是老师/主人，不是被考的人。身份反转带来主动性。
- 让孩子**给小怪兽起名 / 选形象**，建立归属感。
- **可见的成长地图**：把已掌握知识点做成可视化地图/星空，孩子能看到自己"点亮"了多少。
- 进度、成就**属于孩子**，由孩子决定要不要分享给家长（见 6.5）。

### 6.4 主动召回（到点提醒，但不靠家长催）
- 到复习节点时，用**孩子自己的语言/角色**发起召回："你的小怪兽『阿通分』饿了，它想再听你讲一次～"
- 通过微信**订阅消息**（孩子授权的前提下）或在家长端转达，但话术面向孩子、正向。
- **可选**：固定一个轻仪式（如"放学后 5 分钟挑战"），形成习惯而非任务。

### 6.5 家长的正确位置（辅助而非代劳）
- 家长端**只读为主**：看进度、看薄弱知识点、看孩子的讲解片段（孩子可控分享）。
- **明确弱化"代传错题"**：允许家长上传，但产品流程、激励、成就都绑定在"孩子操作"上——家长代劳得不到孩子的成就感反馈，从机制上引导让孩子自己来。
- 给家长**正向话术建议**（"今天他自己讲清了通分，可以夸夸他"），把家长从"监工"变成"啦啦队"。

> 设计红线：**任何环节都不制造"被监视/被考核"的感觉。** 主动性来自成就感、掌控感和好玩，不来自压力。

### 6.6 ⭐ 今日打卡（核心仪式）与奖励机制

把每天的学习收束成一个**清晰、有终点、有奖励**的仪式。孩子最关心的不是"学了多少"，而是"做完了没、能不能玩"。所以打卡要**目标明确、完成即可见奖励**。

#### 6.6.1 怎样才算"今日打卡成功"

**有错题的日子**，需依次完成（每一步都要"做对"才推进）：

**第一步 · 攻克今天的新错题**
1. 上传今天的一道错题（拍照录入）。
2. 把这道错题**重新订正做对**。
3. 系统出一道**类似题**（同知识点、换数字/换情境），孩子**答对**。

**第二步 · 双引擎各出一题**
4. **费曼题**：围绕该知识点，让孩子**讲一遍 / 答一道理解题**，讲清或答对。
5. **艾宾浩斯题**：从历史错题的**复习队列**里取一道**到期复习题**，孩子答对。

> 第一步全过 + 第二步两题全过 = **今日打卡成功** ✅
>
> 说明：第 4 步用费曼检验"是否真懂"（理解维度），第 5 步用艾宾浩斯巩固"以前的薄弱点"（记忆维度）；两者一新一旧，正好对应产品的两个理论内核。
> 边界：若是早期、复习队列里还没有到期题（比如刚开始用、没有历史错题），第 5 步降级为"就当天知识点再出一道间隔练习题"，不让孩子卡住。

#### 6.6.2 没有错题的日子 —— 学习感悟替代

如果当天**没有错题**可上传，不强迫孩子硬凑。提供替代路径：

- 孩子**输入一段"今日学习感悟"**（**支持语音**，符合 7.2；低年级孩子不必打字）。
- 一两句即可（"今天数学课讲了……我觉得……"），系统给一句正向回应。
- 完成感悟 = **今日打卡成功**，同样发放奖励。
- 待确认：若当天虽无新错题、但**复习队列里有到期题**，是否仍要求先完成到期复习再用感悟收尾（建议：要求完成到期题，避免漏复习）——见第 14 章。

#### 6.6.3 奖励：解锁喜欢的游戏时间

打卡成功后，弹出**祝贺动画 + 奖励提示**：

> 🎉 今日打卡完成！你可以去玩 **10 分钟你喜欢的蛋仔派对** 啦～

设计要点：

- 奖励以**"奖励券"**的形式呈现（今日已获得 1 张游戏券）。
- **奖励内容与时长由家长在设置里配置**（默认：10 分钟蛋仔派对；家长可改成别的奖励/时长）。
- 小程序**不启动、不控制**第三方游戏；奖励本质是**"亲子约定 + 家长发放屏幕时间"**。可选：家长端同步收到"孩子今日已打卡，可发放奖励"的提醒，由家长落实。
- 与第 6.2 的怪兽/能量条/徽章叠加：打卡成功**同时点亮今天的怪兽进度**，让"游戏奖励"和"成长成就"一起到账。

#### 6.6.4 设计考量（外部奖励的平衡）

- 用游戏时间这种**外部奖励**见效快，是很好的"启动器"，契合"孩子现在最大的动力是蛋仔"。
- 已知风险：长期单靠物质/游戏奖励可能削弱内在动机。**策略**：用游戏券启动习惯，同时让怪兽点亮、成长地图、徽章承接**内在成就感**；随着习惯养成，奖励节奏由家长逐步把控、自然弱化，而非 App 硬性取消。
- 始终定位为**亲子约定**而非 App 强加，避免家长与孩子因"该不该给玩"产生冲突。

## 7. 关键技术能力

### 7.1 图片分析 / 拍照锁定错题
- 拍照后，自动**检测题目边界**，让孩子**框选/微调**出要录入的那道题，忽略其余内容（其他题、批注、页边）。
- 对框选区域做 **OCR + 版面理解**，提取题干文字、数字、可能的图形/算式。
- 识别题目类型（选择/填空/计算/应用题）以辅助打知识点标签。
- 失败兜底：识别不清时，引导孩子语音补充或重拍，绝不卡死。

### 7.2 语音输入能力
- 全程支持语音：讲解、补充题干、回答测试题、给怪兽命名等。
- 语音转文字（ASR），并保留**原始音频片段**用于费曼讲解分析（语气、流畅度也是理解的信号）。
- 朗读（TTS）：题目、AI 小伙伴的追问可朗读，照顾识字量有限的低年级孩子。

### 7.3 联网搜索 + 同类题生成
- 基于错题的**知识点标签**，优先从**本地题库**（上海小学，见第 9 章）召回同类题。
- 题库覆盖不足时，**联网搜索**补充同类题，并经过**质量/适龄/版权过滤**后再使用。
- 同类题要求："考同一个知识点、换数字/换情境"，难度匹配，避免一模一样的原题。
- 所有 AI 生成/联网获取的题目，使用前需做**正确性校验**（答案与解析自洽）。

### 7.4 AI 能力总览（供后续技术选型）
- 视觉：拍照题目检测、OCR、版面理解。
- 语音：ASR / TTS。
- 文本/推理：费曼讲解分析、追问生成、同类题生成、答案判定、知识点标注。
- 默认优先选用当前最强的多模态大模型；具体供应商与模型在开发阶段确定，本文不锁定。

## 8. 用户旅程与主要页面（信息架构）

### 8.1 核心闭环（孩子端）
```
拍照录题 → 框选锁定 → OCR确认 → 订正做对 → 类似题答对 →
费曼讲解(语音)/费曼题 → 艾宾浩斯到期复习题 → 今日打卡成功 → 解锁游戏券 + 点亮怪兽
                                                  ↑
                          （当天无错题 → 输入学习感悟 → 同样打卡成功）
```
说明：单道错题进入间隔复习队列后，会在第 2/4/7/15/30 天作为"艾宾浩斯到期题"回到每日打卡里复现。

### 8.2 页面清单（v1）
| 页面 | 角色 | 说明 |
|------|------|------|
| 首页 / 今日 | 孩子 | **今日打卡进度条（第一步/第二步）**、待复习怪兽、快速「拍新题」入口 |
| 学习感悟 | 孩子 | 无错题日的替代入口，语音/文字写一段感悟 |
| 打卡成功 / 奖励 | 孩子 | 祝贺动画 + 游戏奖励券 + 今日怪兽点亮 |
| 拍照录题 | 孩子 | 相机 + 框选 + OCR 确认 |
| 费曼讲解 | 孩子 | 语音讲解 + AI 小伙伴追问 |
| 同类题测试 | 孩子 | 做题 + 判定 + 反馈 |
| 我的题库/怪兽图鉴 | 孩子 | 收集、能量条、按知识点聚合 |
| 成长地图/成就 | 孩子 | 已掌握知识点可视化、徽章 |
| 家长端·概览 | 家长 | 只读进度、薄弱知识点、夸夸建议、**今日是否打卡/可发放奖励提醒** |
| 设置 | 双方 | 复习节奏、提醒、**奖励内容与时长配置**、隐私与授权 |

## 9. 题库（上海小学 · 数学 / 语文 / 英语）

- **v1 范围**：上海小学，覆盖**数学、语文、英语**三科（沪教版/上海课标）。
  - 建设顺序建议：**数学先行**（知识点结构最清晰、同类题最好生成），语文/英语紧随其后并行接入。
  - 三科的「同类题」逻辑略有差异，需分别设计：
    - **数学**：换数字/换情境，考同一解题方法（最适合 AI 生成同类题）。
    - **英语**：换词/换句型，考同一语法点或词汇（单词拼写、时态、词组搭配等）。
    - **语文**：字词（拼音/形近字/多音字/听写）、古诗文背默、阅读理解与基础知识点；同类题多为「同知识点换载体」，生成需更强的适龄与正确性把关。
- 结构：题目 → 学科 → 知识点标签（与上海教材章节匹配）→ 年级 → 难度 → 答案 → 解析。
- 来源策略：自建/授权题库为主，联网搜索为补充（需版权与适龄过滤）。可参考官方「沪学习」及沪教版同步资源的知识点组织方式。
- **待确认**：知识点体系采用哪一套标准（沪教版教材章节 / 上海课标）；三科题库的获取与版权方式；语文/英语同类题的生成质量门槛。

## 10. 数据模型（概念级，非建表）

- **用户(孩子)**：昵称、头像、年级、连续打卡、徽章。
- **家长**：关联孩子、查看权限。
- **错题(Question)**：图片、框选区域、OCR文本、题型、知识点标签、录入时间、来源。
- **知识点(KnowledgePoint)**：名称、所属章节、掌握度。
- **讲解记录(Explanation)**：音频、转写文本、AI分析(卡点/理解评分)。
- **复习计划(ReviewSchedule)**：所属错题、下次复习时间、当前间隔档位、历史结果。
- **测试题(Practice)**：题干、来源(题库/AI/联网)、答案、判定结果、类型(类似题/费曼题/艾宾浩斯题)。
- **打卡记录(CheckIn)**：日期、是否成功、完成路径(错题流程/学习感悟)、各步骤结果、发放的奖励券。
- **学习感悟(Reflection)**：日期、音频/文本、AI 回应（无错题日使用）。
- **奖励券(Reward)**：日期、奖励内容(默认蛋仔派对)、时长、是否已由家长发放。

## 11. 隐私、合规与安全（针对未成年人，重要）

- 用户是未成年人，受《未成年人保护法》《个人信息保护法》及微信平台未成年人相关规范约束。
- **监护人知情同意**：注册/采集信息需家长同意；明确告知收集了什么、用途。
- **最小化采集**：只收集必要信息；错题图片/语音的存储期限、用途、是否用于模型训练，需明确并默认保守。
- **内容安全**：联网搜索与 AI 生成内容必须过适龄与安全过滤，杜绝不良信息。
- **数据安全**：图片/语音等敏感数据加密存储、访问受控。
- **可删除**：家长可导出/删除孩子数据。
- **待确认**：是否需要内容安全资质/备案（涉及 UGC + 教育 + 未成年人），开发前法务确认。

## 12. 里程碑（建议）

- **M0 Spec & 设计**（当前）：本文档评审通过 + 关键页面交互原型。
- **M1 录入闭环**：拍照框选 + OCR + 语音，错题能进库。
- **M2 费曼 + 复习**：语音讲解分析、追问、艾宾浩斯队列与召回。
- **M3 测试 + 题库**：同类题（题库优先 + 联网补充）+ 判定 + 反馈。
- **M4 激励系统**：怪兽收集、能量条、成长地图、打卡。
- **M5 家长端 + 合规**：只读概览、订阅消息、隐私同意流程。

每个里程碑可独立验证「孩子是否愿意用」这一核心假设。

## 13. 衡量指标（验证"孩子主动"）

- **主动录入率**：孩子自己录入的错题占比（vs 家长录入）——核心北极星。
- **复习召回率**：到点提醒后孩子主动回来完成复习的比例。
- **知识点掌握率**：进入"掌握"状态的知识点数 / 总知识点数。
- **同类题二次正确率**：同知识点再次出题的答对率（真正掌握的证据）。
- **留存**：次日/7日/30日留存与打卡连续天数。

## 15. 竞品参考与可借鉴经验（中国大陆）

### 15.2 可借鉴的经验 / 技能点

- **拍照→OCR→自动打标签→按知识点聚合**（夸克错题本 / 喵喵机）：录入链路是行业成熟能力，可直接对标其体验标准（识别准、分类自动）。
- **"错题再练 / 同类题推荐"依赖知识图谱**（夸克 / 小猿）：要做好同类题，知识点体系（沪教版章节）是地基，须先建好（呼应第 9 章）。
- **分步骤、引导式解析**（小猿）：比"直接给答案"更利于理解，与我们的费曼追问方向一致，可借鉴其引导话术结构。
- **艾宾浩斯复习节点 + 卡片遮挡复习**（记忆类工具）：复习调度算法与"遮题干自测"交互可直接借鉴。
- **错题报告 / 薄弱点可视化**（夸克）：可作为家长端「只读概览」的参考形态。
- **教材同步与知识点权威性**（沪学习）：上海本地化的关键，借鉴其按年级/课本组织内容的方式。

### 15.3 它们普遍没做、即我们的差异化

1. **重"搜答案/刷题"，轻"是否真懂"** —— 我们用**费曼讲解**做理解检验，治"会抄不会做"。
2. **复习多为"背记卡片"，非"个性化错题复现"** —— 我们把**孩子自己的错题**喂进艾宾浩斯队列，按掌握度动态调度。
3. **驱动力默认来自家长（搜题/打印是家长行为）** —— 我们把**孩子主动**当第一目标（怪兽收集、打卡、游戏奖励券，见第 6 章），这是最大差异点。
4. **工具偏大而全或需硬件** —— 我们用微信小程序，**轻、零安装、单一闭环**，门槛最低。
